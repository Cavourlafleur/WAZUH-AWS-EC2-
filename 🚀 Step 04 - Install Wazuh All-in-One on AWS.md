# 🚀 Step 04 - Install Wazuh All-in-One on AWS

## 🎯 Goal

Install the Wazuh central server on the AWS Ubuntu EC2 instance.

This creates an all-in-one Wazuh deployment with:

- Wazuh Manager 🛡️
- Wazuh Indexer 📚
- Wazuh Dashboard 📊
- Filebeat 📦

---

## 🧠 What This Step Does

This step installs the full Wazuh central platform on one server. Wazuh Quickstart says the installation assistant deploys the Wazuh server, indexer, and dashboard on the same host [web:44].

### Kid meaning

This is like building the security office, the filing cabinet, and the control screen all in one place 🏢✨

---

## ✅ Before You Start

Make sure you are already inside the AWS Ubuntu server.

You should see something like:

```bash
ubuntu@ip-172-31-xx-xx:~$
```

That means you are logged into the AWS EC2 server and ready to install.

---

## 🛠️ Commands Used

### 1️⃣ Install curl if needed

```bash
sudo apt install -y curl
```

| Part | Meaning |
|---|---|
| `sudo` | Run as administrator |
| `apt` | Ubuntu package manager |
| `install` | Install software |
| `-y` | Automatically answer yes |
| `curl` | Tool used to download files from the internet |

### ✅ Purpose

Install the download tool needed to fetch the Wazuh installer.

### 👶 Kid meaning

Install the tool that helps us download the Wazuh installer.

---

### 2️⃣ Download the Wazuh installer

```bash
curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh
```

| Part | Meaning |
|---|---|
| `curl` | Downloads something from the internet |
| `-s` | Silent mode |
| `-O` | Save using the same filename from the website |
| `https://packages.wazuh.com/4.14/wazuh-install.sh` | Official Wazuh installer script |

### ✅ Purpose

Download the Wazuh installation assistant onto the AWS server.

### 👶 Kid meaning

Download the Wazuh installer script to this server.

Wazuh Quickstart shows this exact installer workflow for the all-in-one setup [web:44].

---

### 3️⃣ Confirm the file downloaded

```bash
ls -lh wazuh-install.sh
```

| Part | Meaning |
|---|---|
| `ls` | List files |
| `-l` | Long/detail view |
| `-h` | Human-readable file size |
| `wazuh-install.sh` | The installer file we downloaded |

### ✅ Purpose

Make sure the installer file is present.

### 👶 Kid meaning

Show me the installer file and its size.

---

### 4️⃣ Run the Wazuh all-in-one installer

```bash
sudo bash ./wazuh-install.sh -a
```

| Part | Meaning |
|---|---|
| `sudo` | Run as administrator/root |
| `bash` | Run the script using Bash |
| `./wazuh-install.sh` | Run the installer file in the current folder |
| `-a` | All-in-one install |

### ✅ Purpose

Install the complete Wazuh central platform on this AWS server.

### 👶 Kid meaning

Install the whole Wazuh security system on this server.

Wazuh Quickstart confirms that running the assistant with `-a` installs the platform and then prints the web access credentials when finished [web:44].

---

## 🔐 Save the Admin Password

At the end, Wazuh should print something like:

```text
You can access the web interface:
https://<WAZUH_DASHBOARD_IP_ADDRESS>
User: admin
Password: <ADMIN_PASSWORD>
```

### Important

- Save the username and password in a private password manager or private note 🔒
- Do **not** commit the password to GitHub 🚫
- Do **not** place it in README files or public docs 🚫
- Do **not** screenshot it into public documentation 🚫

### Safe GitHub wording

```text
Admin credentials were generated by the installer and stored securely outside the repository.
```

Wazuh Quickstart says the installer output shows the dashboard URL, the `admin` user, and the generated password after installation [web:44].

## 6️⃣ Check major services

Run these one by one:

```bash
sudo systemctl status wazuh-manager --no-pager
```

```bash
sudo systemctl status wazuh-indexer --no-pager
```

```bash
sudo systemctl status wazuh-dashboard --no-pager
```

```bash
sudo systemctl status filebeat --no-pager
```

| Part | Meaning |
|---|---|
| `systemctl` | Linux service manager |
| `status` | Show whether a service is running |
| `--no-pager` | Show output directly |
| `wazuh-manager` | Wazuh manager service |
| `wazuh-indexer` | Wazuh indexer service |
| `wazuh-dashboard` | Wazuh dashboard service |
| `filebeat` | Helper that forwards alerts/events |

### ✅ Purpose

Confirm the main Wazuh services are active.

### 👶 Kid meaning

Check if each Wazuh part is alive.

Look for:

```text
active (running)
```

That is good ✅

## 🚨 If Something Fails

Do not panic.

If install fails or a service is not running, check the logs:

```bash
sudo journalctl -u wazuh-manager -n 50 --no-pager
```

```bash
sudo journalctl -u wazuh-indexer -n 50 --no-pager
```

```bash
sudo journalctl -u wazuh-dashboard -n 50 --no-pager
```

| Part | Meaning |
|---|---|
| `journalctl` | Shows Linux service logs |
| `-u` | Choose a service/unit |
| `-n 50` | Show last 50 lines |
| `--no-pager` | Print directly |

### ✅ Purpose

Show recent logs if a Wazuh service fails.

### 👶 Kid meaning

Read the last messages from the broken service so we can find the problem.

## 🔐 Security Notes

Do not leave admin access open to the whole internet.

### Recommended security group direction

| Port | Purpose | Source |
|---|---|---|
| 22/TCP | SSH | My IP only |
| 443/TCP | Wazuh Dashboard | My IP only |
| 1514/TCP | Agent communication | Known agents only |
| 1515/TCP | Agent enrollment | Temporary or known agents only |
| 55000/TCP | Wazuh API | Do not expose publicly |
